🎤 33. Voice, Video, Soundboard & DAVE/E2EE 0 of ~45 — IGNORED (roadmap) ↗ docs

Discord's voice/video subsystem. Currently out of scope (see VOICE_PENDING.md). This section lists all features with direct links to the official doc — just follow the order to implement.

Note: Voice on Discord is composed of four independent subsystems: (1) Voice Resource REST — states and regions; (2) Voice Gateway — second WebSocket per voice session, with 30+ opcodes including the DAVE/E2EE protocol; (3) RTP/UDP transport — Opus + encryption (AEAD AES-256-GCM or XChaCha20-Poly1305); (4) Soundboard — REST endpoints + events. Implement in this order: REST → Gateway → UDP → Soundboard → DAVE.
